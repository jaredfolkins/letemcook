LEMC Environment Variables

LEMC uses environment variables for configuration and context.

System-Provided Context Variables

These are often set by the LEMC execution environment to provide context to the job:

  • LEMC_UUID: A unique identifier, often for the specific recipe instance (Currently: %%LEMC_UUID%%)
  • LEMC_SCOPE: The execution scope (e.g., individual, team) (Currently: %%LEMC_SCOPE%%)
  • LEMC_USER_ID: The ID of the initiating user (Currently: %%LEMC_USER_ID%%)
  • LEMC_USERNAME: The username of the initiating user (Currently: %%LEMC_USERNAME%%)
  • LEMC_RECIPE_NAME: The name of the executed recipe (Currently: %%LEMC_RECIPE_NAME%%)
  • LEMC_PAGE_ID: The ID of the associated web page (if applicable) (Currently: %%LEMC_PAGE_ID%%)
  • LEMC_STEP_ID: The specific step within the action (Currently: %%LEMC_STEP_ID%%)
  • LEMC_HTTP_DOWNLOAD_BASE_URL: Base URL for artifact downloads (Currently: %%LEMC_HTTP_DOWNLOAD_BASE_URL%%)

Developer Defined Recipe Variables

Some example environment variables are configured by the developer when the recipe is created:

  • USER_DEFINED_PUBLIC_ENV_VAR: (Example Public Var) Specifies the primary recipe/build file (Currently: %%USER_DEFINED_PUBLIC_ENV_VAR%%)
  • USER_DEFINED_PRIVATE_ENV_VAR: (Example Private Var) Specifies the primary recipe/build file (Currently: %%USER_DEFINED_PRIVATE_ENV_VAR%%)

Container Mount Points

The container has several directories mounted from the host system:

  • /lemc/public: Directory for public artifacts. Files here are available for download.
  • /lemc/private: Directory for private artifacts. Files here are kept here for state or for future use. Not available for download.
  • /lemc/global: Directory for globally shared resources for a cookbook/app.
  • /lemc/shared: Directory for shared resources (potentially writable, often admin-only).
